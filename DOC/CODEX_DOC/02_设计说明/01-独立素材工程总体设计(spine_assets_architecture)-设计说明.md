# 独立素材工程总体设计

## 1. 目标

建立一个独立于主游戏工程的 `Spine` 素材工作仓库，使其可以：

1. 读取主工程角色目录
2. 输出标准化素材产物
3. 允许未来与主工程并行开发

## 2. 分层

本工程分为四层：

1. `Input Sync`
   - 读取主工程 `player / enemy` 目录
2. `Asset Workspace`
   - 管理分件图、Rig 和 `Spine` 工程源文件
3. `Export Pipeline`
   - 导出 `Spine bundle`
4. `Validation`
   - 校验导出物和 manifest

## 3. 当前首版范围

首版只落地：

1. 仓库骨架
2. 输入同步脚手架
3. 契约文档
4. 导出目标格式定义

首版不落地：

1. 正式 `Spine` 源文件
2. 角色动作资产
3. 主工程接入代码
