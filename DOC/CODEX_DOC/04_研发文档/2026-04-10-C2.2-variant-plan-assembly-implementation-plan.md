# Variant Plan Assembly Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make blackbox-generated `variant_plan.json` become package/export truth so multiple skins or variants under one `presentationId` are actually assembled, validated, and previewed.

**Architecture:** Keep the existing filesystem-first pipeline. `collect_blackbox_results` becomes the single merge point from blackbox plan artifacts into `workspace/packages/<presentationId>/variants/*/variant.json`, then export and preview consume those merged manifests without inventing parallel state. Backward compatibility stays intact by treating missing plan fields as “preserve package defaults”.

**Tech Stack:** Node.js ESM, JSON manifests, existing npm scripts, HTML preview generation, node:test
