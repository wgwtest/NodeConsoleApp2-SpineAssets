# Workflow Acceptance Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single acceptance page that shows the full sample flow from input image and supplemented request to blackbox planning, package collection, export output, and the derived paperdoll assembly object.

**Architecture:** Reuse existing filesystem artifacts as the source of truth. Add one new workflow preview builder that reads `workspace/requests`, `workspace/blackbox_jobs`, `workspace/packages`, and `workspace/exports/request_driven_bundle`, aligns them by `characterRequestId` and `presentationId`, then renders a single HTML timeline plus a structured report JSON. The final step derives a runtime-facing paperdoll object snapshot from package and export data without pretending that a full runtime assembler already exists.

**Tech Stack:** Node.js ESM, node:test, existing validators, filesystem JSON artifacts, static HTML preview generation
