# Hybrid Component Decomposition Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a real, reviewable `original art -> visible slot components` pipeline, then layer optional completion candidates on top without confusing generated guesses with factual extraction.

**Architecture:** Keep the existing Node.js blackbox/package/export production line as the source of truth. Add one new hybrid decomposition provider that reuses current planning artifacts, delegates visible extraction to a focused Python helper stack, writes masks/overlays/visible components back into the existing job artifact tree, and only treats completion output as optional candidate data. Gates and previews are upgraded so exports depend on real visible extraction rather than placeholder render copies.

**Tech Stack:** Node.js ESM, node:test, filesystem JSON contracts, Python 3.12 helper CLI, local grounding + segmentation models, Pillow image postprocess, existing HTML preview generation
