# WBS L2：3.3 拆件加工

## 1. 目标

把 `workspace/requests/<characterRequestId>/` 中的角色原画、描述和参考输入，转换为可被后续角色包收口使用的组件草稿和加工证据。

## 2. 子节点

1. `3.3.1 原图拆件与过程留痕`

## 3. 前置条件

1. `3.2` 已完成任务建档
2. `package_manifest / variant.json / character_manifest` 已固定基础字段边界

## 4. 预期输出

1. 组件拆件加工的接口文件和状态留痕规则
2. `components/` 目录下组件描述、来源和产出规则
3. 给 `3.4 角色包收口` 使用的组件草稿和证据文件

## 5. 关闭标准

1. 任一 request 的原画输入都能留下组件加工过程证据
2. 输出物能够被 `3.4 角色包收口` 继续消费
3. 人工加工和云端加工都能落到统一目录结构
