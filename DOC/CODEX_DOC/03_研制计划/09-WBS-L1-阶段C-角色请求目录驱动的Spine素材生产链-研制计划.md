# WBS L1：3 正式生产主线

## 1. 目标

把本仓库正式主线固定为六个线性步骤，使其不再依赖内部术语叙事，而是能够让人直接看懂每一步的输入、处理和输出。

## 2. 子节点

1. `3.1 输入接收`
2. `3.2 任务建档`
3. `3.3 拆件加工`
4. `3.4 角色包收口`
5. `3.5 交付包导出`
6. `3.6 运行验收`

## 3. 前置条件

1. `1` 已冻结双仓库接口归属和基础 manifest 契约
2. `2` 可作为 bundle/manifest 回归验证基线继续复用

## 4. 预期输出

1. `3.1` 输入接收的目录契约
2. `3.2` 任务建档的 job 冻结与回放规则
3. `3.3` 拆件加工的接口、证据和补图能力
4. `3.4` 角色包收口的组件完备性约束
5. `3.5` 交付包导出的正式 bundle 与 manifest
6. `3.6` 运行验收的预览页、运行快照与人工验收入口

## 5. 关闭标准

1. 六个主流程步骤都能被清楚地区分输入、处理和输出
2. 最小 request fixture 可以完成从输入接收到交付包导出的完整闭环
3. `2` 回归链不被破坏
4. 组件加工阶段能够在 package 内留痕，并形成后续装配约束
