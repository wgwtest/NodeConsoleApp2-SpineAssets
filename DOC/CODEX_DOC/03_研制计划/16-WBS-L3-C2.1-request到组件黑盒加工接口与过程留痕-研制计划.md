# WBS L3：C2.1 request 到组件黑盒加工接口与过程留痕

## 1. 目标

把 request 输入目录中的原画、描述和参考资料，转换为本仓库内部可追踪的组件加工工单与过程记录，使人工黑盒加工阶段具备可审计、可回放的最小证据。

## 2. 预期输出

1. 组件加工工单或状态文件规则
2. `components/<componentId>/` 下的来源、状态、产出描述约定
3. request 源文件到组件产物之间的 lineage 记录

## 3. 进入条件

1. `C1.1` 已固定 request 与 package 的目录边界
2. `source/request_snapshot.json` 已作为 request 拷贝基线存在

## 4. 关闭标准

1. request 内的原画和说明可以映射到至少一个组件加工记录
2. 每个组件目录都能记录来源文件、当前状态和产出文件
3. 只看 package 内留痕文件，就能知道哪些组件仍处于黑盒加工中
