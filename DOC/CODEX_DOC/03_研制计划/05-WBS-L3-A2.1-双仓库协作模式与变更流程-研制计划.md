# WBS L3：A2.1 双仓库协作模式与变更流程

## 1. 工作目标

固定两仓库的职责分工、镜像文档规则、breaking change 流程和并行开发边界。

## 2. 工作内容

1. 固定主工程与素材工程的职责分工
2. 固定镜像文档必须同步更新的主题
3. 固定 breaking change 的触发条件与双仓库联动流程
4. 固定并行开发时“可推进 / 不可私自扩写”的边界

## 3. 成果物

1. 协作模式设计说明
2. 主工程镜像规则文档
3. 双仓库对应 issue

## 4. 验收方法

1. 任意只打开单仓库，也能判断本仓库负责什么、不负责什么
2. 能明确知道 breaking change 发生时要在两仓库同时做什么

## 5. 当前状态

1. `已自测`
2. `待人工验收`
