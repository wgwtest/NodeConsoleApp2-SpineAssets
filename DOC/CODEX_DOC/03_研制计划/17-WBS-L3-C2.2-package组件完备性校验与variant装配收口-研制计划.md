# WBS L2：3.4 角色包收口

## 1. 目标

把拆件输出物、variant 依赖和 package 目录收成一套统一角色包，使后续交付包导出能够直接消费。

## 2. 预期输出

1. 组件完备性状态与最小字段约定
2. `variant.requiredComponents` 与组件目录之间的一致性校验
3. `readiness / reviewStatus / evidenceFiles` 的收口门槛
4. 角色包预览与收口报告

## 3. 进入条件

1. `3.3.1` 已定义组件加工留痕接口
2. `3.5` 已固定正式导出的目标结构

## 4. 关闭标准

1. 缺失组件、组件加工未完成、组件 evidence 不完整或组件状态不允许收口时，角色包会稳定失败
2. 同一 `presentationId` 下多个 `variant` 的组件覆盖关系可以被机器报告出来
3. 角色包收口通过后，正式 export 行为与 `3.5` 保持兼容
