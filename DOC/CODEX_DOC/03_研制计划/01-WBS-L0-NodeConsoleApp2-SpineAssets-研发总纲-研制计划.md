# WBS L0：NodeConsoleApp2-SpineAssets 研发总纲

## 1. 总目标

建立独立的 `Spine` 素材制作工程，使其可以向 `NodeConsoleApp2` 稳定输出战斗展示素材产物。

## 2. WBS 主树

1. `A1`
   - 项目底座与契约建立
2. `B1`
   - 样本角色导入、Rig 与导出验证
3. `C1`
   - 主工程导入接入与 bundle 交付规范

## 3. 节点依赖

1. `A1 -> B1 -> C1`
2. `A1` 是两个工程后续并行推进的前置边界建立节点
3. `B1` 负责证明素材工作流在样本角色上跑通
4. `C1` 负责把导出产物纳入主工程消费链

## 4. 当前活跃节点

1. `A1`

## 5. 阶段关闭标准

1. `A1`
   - 独立仓库可运行
   - 输入输出契约成文
   - 同步脚手架能读取主工程真实目录
2. `B1`
   - 至少一个玩家样本和一个敌人样本完成导入、Rig、导出、预览验证
3. `C1`
   - 主工程能够按约定消费 bundle，且失败路径可诊断
